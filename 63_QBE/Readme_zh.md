# 第 63 部分：QBE 后端

我在 2019 年底留下了编译器的最后一个版本，当时计划研究一种改进的寄存器分配方案。然而，在此期间发生了很多事情，包括 2020 年中期的个人悲剧。

就在几周前（2021 年 12 月中旬），我遇到了一个由 Quentin Carbonneaux 编写的名为
[QBE](https://c9x.me/compile/) 的项目。
该工具描述了一种中间语言，非常适合我的编译器输出。这种中间语言会被翻译
成真正的汇编代码。此外，QBE 提供并执行以下功能：

 * 基于 SSA 的中间语言
 * 带提示的线性寄存器分配器
 * 复制消除
 * 稀疏条件常量传播
 * 死指令消除
 * 小栈槽的寄存器化
 * 借助 SSA 形式的分离溢出器和寄存器分配器
 * 基于循环分析的智能溢出启发式

从本质上讲，QBE 执行了许多编译器应该做的后端寄存器和代码优化
工作。而且，由于 Quentin 已经编写了这些代码，我决定丢弃现有的 x86-64 代码生成器，
转而编写一个输出 QBE 中间语言的代码生成器。

这就是当前版本的 `acwj` 编译器。它仍然通过了三重测试。然而，使用 QBE 后端
输出的汇编代码大小约为版本 62 编译器输出的一半。

如果你想尝试这个版本的 `acwj` 编译器，你需要
下载并编译 [QBE](https://c9x.me/compile/)，并将
`qbe` 可执行文件安装在你的 $PATH 中的某个位置。`acwj` 编译器将输出
中间代码到一个以 `.q` 为后缀的文件。然后它调用
`qbe` 将其翻译成汇编代码，并继续执行通常的步骤来汇编和链接该代码。

那么，让我们开始吧！

## QBE 中间语言

现在，我*真的*想做的事情是向你解释 QBE 如何实现
[静态单赋值形式](https://en.wikipedia.org/wiki/Static_single_assignment_form)、寄存器分配、死代码消除等。但是，
我自己对这些东西还没有很好的理解。也许有人
可以浏览 QBE 源代码，并用我解释 `acwj` 编译器的方式来解释它的工作原理。

相反，我将稍微介绍一下 QBE 使用的中间语言，
并解释我是如何在 `cg.c`（我的新代码生成器）中针对这个语言的。

## 临时位置，而非寄存器

QBE 中间语言是一种抽象语言，而不是
真实 CPU 的汇编语言。因此，它不必具有
相同的限制，例如固定的一组寄存器。

相反，有无限数量的*临时*位置，每个
位置都有自己的名称。全局可见的临时位置
以 `$` 字符开头，而仅在函数内可见的
临时位置以 `%` 字符开头。

临时位置不需要预先定义：它们可以动态创建。然而，创建时，
每个临时位置都被定义为具有几种*类型*之一。这些类型（及其
后缀）是：

 * 8 位字节（*b*）
 * 16 位半字（*h*）
 * 32 位字（*w*）
 * 64 位长字（*l*）

QBE 还提供**单**精度浮点、**双**精度浮点
以及定义聚合类型的方法。我在 `acwj` 中没有使用这些，
因此不会讨论它们。但是，你可以在
[QBE 中间语言参考文档](https://c9x.me/compile/doc/il.html) 中阅读相关内容。

局部临时变量可以通过在汇编语言中执行通常的操作来创建。一些例子是：

```
  %b0 =w copy 5              # 创建 %b0 作为字临时变量并
                             # 用值 5 初始化它
  %fred =w add %c, %d        # 将两个临时变量相加并存储在
                             # %fred 字临时变量中
  %p =h call ntohs(h %foo)   # 用 %foo 临时变量的值调用 ntohs()
                             # 并将半字结果保存在 %p 临时变量中
```

## 类型混合

每个临时变量都有类型。这意味着你确实需要做一些
类型之间的转换。例如，你不能这样做：

```
  %x =w copy 5               # int x = 5;
  %y =l copy %x              # long y = x;
```

当你想将较小类型的值扩展为较大类型时，
你需要确定较小类型是*有符号*还是*无符号*。
例子：

```
  %x =w copy -5   # int x = -5;               32 位值 0xfffffffb
  %y =l extsw %x  # long y = x;               0xfffffffffffffffb
  %z =l extuw %x  # long z = (unsigned) x;    0x00000000fffffffb
```

另一方面，你可以将一个宽值存储到较小的临时
位置；QBE 只需截断最高有效位。

## 我们的第一个例子

以下是这个 C 程序的手动翻译：

```c
#include <stdio.h>

int main()
{
  int x= 5;
  long y= x;
  int z= (int)y;
  printf("%d %ld %d\n", x, y, z);
  return(0);
}
```

成 QBE 中间语言：

```
data $L19 = { b "%d %ld %d\n" }
export function w $main() {
@L20
  %x =w copy 5
  %y =l extsw %x
  %z =w copy %y
  call $printf(l $L19, w %x, l %y, w %z)
  ret 0
}
```

还有一些我还没有描述的东西。字符串字面量
`"%d %ld %d\n"` 作为一系列**b**ytes 存储在一个名为
`$L19` 的全局临时变量中。从技术上讲，`$L19` 是
字符串第一个字节的地址。

`main()` 被定义为一个非局部函数（因此带有 `$`），它返回一个
32 位**w**ord。`export` 关键字表示该函数
在此文件外部可见。

`@L20` 是一个标签，就像普通汇编标签一样。QBE 要求每个
函数有一个起始标签。

最后，`ret` 操作从函数返回。每个函数只能有一个
`ret` 操作。它必须是函数中的最后一行，并且
赋予它的任何值必须与函数的类型匹配。

## acwj 的 QBE 输出

现在让我们看看 `acwj` 如何将上述 C 程序编译成 QBE
中间语言：

```
export function w $main() {
@L20
  %.t1 =w copy 5
  %x =w copy %.t1               # x = 5;
  %.t2 =w copy %x
  %.t3 =l extsw %.t2
  %y =l copy %.t3               # y = x;
  %.t4 =l copy %y
  %.t5 =w copy %.t4
  %z =w copy %.t5               # z = (int) y;
  %.t6 =w copy %z
  %.t7 =l copy %y               # 将参数放入"寄存器"
  %.t8 =w copy %x
  %.t9 =l copy $L19             # 调用 printf()，获取结果
  %.t10 =w call $printf(l %.t9, w %.t8, l %.t7, w %.t6, )
  %.t11 =w copy 0
  %.ret =w copy %.t11           # 将返回值设为 0
  jmp @L18
@L18
  ret %.ret
}
```

相当不理想，对吧？！`acwj` 仍然认为变量如 `x` 和 `y`
驻留在内存中，而"寄存器"必须用于在变量之间移动
数据。我使用以 ".t" 开头的临时名称，这样就不会与
实际的 C 变量名冲突。

`return(0)` 被翻译成将值复制到
`%.ret` 临时变量然后跳转到函数最后一行的代码。
显然，这里不需要那个跳转。

总的来说，`acwj` 输出的代码效率相当低。这就是
我想在 `acwj` 中添加一些优化的原因。好处是，现在
QBE 在死代码消除和代码优化方面做得很好。以下是 QBE 对上述中间代码执行的 x86-64 翻译：

```asm
.text
.globl main
main:
        pushq %rbp
        movq %rsp, %rbp                 # 设置帧和栈指针
        movl $5, %ecx                   # 将 5 复制到三个参数中
        movl $5, %edx
        movl $5, %esi
        leaq L19(%rip), %rdi            # 加载字符串的地址
        callq printf                    # 调用 printf()
        movl $0, %eax                   # 设置 main() 的返回值
        leave
        ret                             # 并从 main() 返回
```

可爱。一切都存储在寄存器中，没有任何局部变量使用栈。

## 带地址的局部变量

QBE 在尽可能将数据保存在寄存器方面做得很好。但
有些时候这是不可能的。当我们需要获取
变量的地址时，例如

```c
int main()
{
  int x= 5;
  int *p = &x;
  printf("%d %lx\n", x, (long)p);
  return(0);
}
```

`x` 变量肯定需要存储在内存中，这样我们才能
获取存储在 `p` 中的地址。为此，我们使用 QBE 操作
来分配和访问内存：

```
export function w $main() {
@L20
  %x =l alloc8 1                # 为 x 分配 8 字节
  %.t1 =w copy 5
  storew %.t1, %x               # 将 5 存储为 x 中的 32 位值
  %.t2 =l copy %x               # 获取 x 的地址
  %p =l copy %.t2
  %.t3 =l copy %p
  %.t5 =w loadsw %x             # 获取 x 处的 32 位值
  %.t6 =l copy $L19
  %.t7 =w call $printf(l %.t6, w %.t5, l %.t3)
  %.t8 =w copy 0
  %.ret =w copy %.t8
  jmp @L18
@L18
  ret %.ret
}
```

`%x` 现在被当作指向栈上八个字节的指针。我选择
以八字节为一组进行分配，因为这有助于保持
8 字节长整型和指针正确对齐。我们现在需要使用
`store` 和 `load` 操作来写入和读取 `%x` 指向的内存位置。

上述中间代码被 QBE 翻译为：

```
main:
        pushq %rbp
        movq %rsp, %rbp
        subq $16, %rsp                  # 在栈上分配空间
        movl $5, -8(%rbp)               # 将 5 作为 x 存储在栈上
        leaq -8(%rbp), %rdx             # 获取 x 的地址
        movl $5, %esi                   # 优化：使用字面量 5
        leaq L19(%rip), %rdi            # 而不是访问栈
        callq printf
        movl $0, %eax
        leave
        ret
```

这是一个相当理想的 `acwj` 中间语言翻译！

## QBE 和字符

QBE 不将 8 位字节或 16 位半字作为主要类型：没有
字节或半字临时位置。相反，这些必须存储在栈上或堆上。因此，`acwj` 编译这个 C 代码：

```c
int main()
{
  char x= 65;
  printf("%c\n", x);
  return(0);
}
```

为：

```
export function w $main() {
@L20
  %x =l alloc4 1                        # 在栈上分配 4 字节
  %.t1 =w copy 65
  storew %.t1, %x                       # 将 65 存储为一个 16 位字
  %.t2 =w loadub %x                     # 重新加载为 8 位无符号字节
  ...
}
```

# 比较和条件跳转

QBE 有指令来比较两个临时变量，如果比较为真则将
第三个临时变量设置为 1，否则设置为 0。指令是：

 * `ceq` 等于
 * `cne` 不等于
 * `csle` 有符号小于或等于
 * `cslt` 有符号小于
 * `csge` 有符号大于或等于
 * `csgt` 有符号大于
 * `cule` 无符号小于或等于
 * `cult` 无符号小于
 * `cuge` 无符号大于或等于
 * `cugt` 无符号大于

后跟两个参数的类型字母。因此，C 代码：

```c
  int x= 5;
  int y= 6;
  int z= x>y;
```

可以编译成中间代码：

```
  %x =w copy 5
  %y =w copy 6
  %z =w csgtw %x, %y
```

QBE 只有一条条件跳转指令：`jnz`。当命名临时
位置非零时，`jnz` 跳转到第一个标签。否则它跳转到
第二个标签。`jnz` 必须有两个标签。

使用这个，我们可以翻译这个 C 代码：

```c
  if (5>6)
    z= 100;
  else
    z= 200;
```

为：

```
@L19
  %.t1 =w csgtw 5, 6            # 比较 5>6，将结果存储在 %.t1 中
  jnz %.t1, @Ltrue, @Lfalse     # 如果为真跳转到 @Ltrue，否则跳转到 @Lfalse
@Ltrue
  %z =w copy 100                # 将 z 设为 100 并使用
  jmp @L18                      # 绝对跳转指令 jmp 跳过
@Lfalse
  %z =w copy 200                # 将 z 设为 200
@L18
  ...
```

使用比较指令和 `jnz`，我们可以实现 IF、FOR 和
WHILE 结构。

## 结构体和数组

这些非常简单。要访问
结构体中的字段，获取基地址并加上
字段的偏移量。对于数组元素，我们需要
将元素的索引乘以每个元素的大小。因此这个 C 代码：

```c
struct foo {
    int field1;
    int field2;
} x;

int main() {
  x.field2= 45;
  return(0);
}
```

由 `acwj` 编译为：

```
export data $x = align 8 { l 0 }
export function w $main() {
@L19
  %.t1 =w copy 45
  %.t2 =l copy $x               # 获取 x 的基地址
  %.t3 =l copy 4
  %.t2 =l add %.t2, %.t3        # 加上 4
  storew %.t1, %.t2             # 在这个地址存储 45
  ...
}
```
