# 翻译规范与指南

本文档为 ACWJ 项目的中文翻译提供统一标准。

---

## 翻译原则

### 1. 准确性优先

- 忠实传达原文含义
- 不添加原文中没有的信息
- 不过度解释，除非有助于理解

### 2. 一致性至上

- **核心术语必须查对术语表**，使用统一的中文译法
- 同一术语在全文中必须使用相同翻译
- 交叉引用章节时，使用中文标题

### 3. 可读性第二

- 中文表达应自然流畅
- 避免翻译腔（机翻译的生硬感）
- 长句可适当拆分

### 4. 保留技术细节

- 代码注释应翻译
- 变量名、函数名保留英文
- 命令行输出保留英文

---

## 源代码注释翻译规范

### 需要翻译的内容

源代码（.c, .h 文件）中的注释需要翻译：

| 注释类型 | 示例 | 处理方式 |
|---------|------|---------|
| 功能描述注释 | `// Lexical scanning` | 翻译为中文 |
| 函数说明注释 | `// Get the next character from the input file.` | 翻译为中文 |
| 行内注释 | `// Use the character put back if there is one` | 翻译为中文 |
| 版权声明 | `// Copyright (c) 2019 Warren Toomey, GPL3` | **不翻译**，保持原文 |

### 翻译流程

1. 读取源文件（.c 或 .h）
2. 翻译所有非版权注释
3. 保留代码结构不变
4. 保存为 `filename_zh.c` 或 `filename_zh.h`

### 术语一致性

源代码翻译时，同样需要查对 `GLOSSARY.md` 中的术语表，确保：
- token → 词法单元
- AST → 抽象语法树
- parser/parsing → 解析器/解析
- scanner/lexical → 扫描器/词法

## 格式规范

### 标题处理

- `# Part X: Title` → `# 第 X 章：标题`
- `## Section` → `## 小节名称`
- 代码块中的标题不需要翻译

### 链接处理

- 章节间链接：`[Next step](../02_Parser/Readme.md)` → `[下一步](../02_Parser/Readme.md)`
- 外部链接保留原样

### 代码块处理

- 代码注释应翻译
- 示例输入/输出中的注释应翻译
- 代码本身不翻译

### 列表处理

- 无序列表项如为主动句，翻译时注意语序调整
- 有序列表项保持编号对应

---

## 术语使用规则

### 必查术语表

翻译前**必须**查阅 `GLOSSARY.md`，确认术语已有翻译。

### 术语表未收录的术语

1. 根据上下文推断合适翻译
2. 首次出现时，在当页脚注或行内注明英文原词
3. 事后更新 GLOSSARY.md

### 常见术语翻译决策

| 情况 | 处理方式 |
|-----|---------|
| 已有标准译法 | 使用标准译法 |
| 术语表已有 | 严格遵循术语表 |
| 口语化表达 | 可用"这个"、"那个"等 |
| 专有名词（人名）| 保留英文，可附中文 |

## Markdown 格式注意事项

### 链接路径

章节内链接使用相对路径，翻译后仍有效：

```markdown
[下一步](../02_Parser/Readme.md)
```

### 图片路径

```markdown
![](Figs/parsing_steps.png)
```

图片不翻译路径，但翻译 alt 文本（如果有）。

### 强调和代码

```markdown
- `struct` → struct（保留代码格式）
- *token* → token（保留斜体，如果这是术语）
- ` ` `c → 保持代码块中的英文
```

## 特殊内容处理

### 版权声明

版权部分**不翻译**，保持原文。

### 命令行示例

```bash
$ make
cc -o parser -g expr.c interp.c main.c scan.c tree.c
```

命令不翻译。输出可以翻译或保留英文，**保持一致即可**。

### 常见错误消息

| 原文 | 推荐翻译 |
|-----|---------|
| Unrecognised character | 无法识别的字符 |
| syntax error | 语法错误 |
| Unknown token | 未知的 Token |

- [ ] 是否翻译了所有.c和.h文件
- [ ] 所有核心术语已查表并一致使用
- [ ] 标题已翻译为中文
- [ ] 章节间链接文字已翻译
- [ ] 代码注释已翻译
- [ ] 无翻译腔明显的句子
- [ ] 格式（标题级别、列表、代码块）保持正确
- [ ] 更新 PROGRESS.md 中该章节状态

## 更新术语表

当遇到新术语时：

1. 记录英文原词
2. 给出中文翻译
3. 提供简要说明或例句
4. 提交到 GLOSSARY.md
